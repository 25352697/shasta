Running an assembly: simplified procedure for small runs

This section describes a simplified procedure to use the shasta software for a small assembly run, perhaps for testing or experimentation. Keep in mind, however, that the shasta software is not designed or optimized for these small assemblies. For a large run for which performance is essential, you can use the complete procedure described here.

Setting up a run directory

Simply create a new directory to contain your run, cd to that new directory, and run shasta-install/bin/SetupSmallRunDirectory.py. This will create some required directories and symbolic links that which cause all shasta binary data created during the run to be stored on disk.

Next, you need to create a configuration file where the shasta software will find values for various assembly parameters. This configuration file must be named shasta.conf. Sample configuration files are available in the shasta-install/conf directory. You can select one of them, copy it to the run directory under the new name shasta.conf, and edit it to customize it as necessary for your run.

Running the assembly

You can now just invoke shasta-install/bin/RunAssembly.py, passing as arguments the names of your input FASTA files, and the assembly will start.
